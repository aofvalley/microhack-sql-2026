Idle
Security note: this tool runs scripts\deploy.ps1 / cleanup.ps1 using the container's or host's az login. Protect it with a password (APP_PASSWORD) and publish it only behind a trusted boundary (private network / VPN / TLS-terminating proxy).
This tool covers three operations:
  • Initial deployment — create N students starting at index 1.
  • Add students / extra infra — provision more students without touching the existing ones. Set Number of students to how many to add and Start index to the next free index (e.g. you already have 1–20, set start index 21 and 1 student to add user21). Use Detect next free index to fill it in automatically. Resource names derive from the index, so existing students are never overwritten.
  • Clean up — bottom panel: delete every lab resource group (and optionally the Entra users).
