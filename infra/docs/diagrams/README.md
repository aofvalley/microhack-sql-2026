# Diagrams

Professional Azure architecture diagrams for the student documentation, built with
[draw.io](https://www.drawio.com/) and the **official Microsoft Azure architecture icons**.

| Source (`.drawio`) | Rendered PNG | Used in |
| --- | --- | --- |
| `architecture.drawio` | `../images/architecture.png` | [architecture.md](../architecture.md), [introduction.md](../introduction.md) |
| `access-flow.drawio` | `../images/access-flow.png` | [access-guide.md](../access-guide.md) |

## How they are built

The `.drawio` files are generated by `build-diagrams.py`, which embeds the official Azure icon
SVGs from [`icons/`](icons/) as data URIs so the diagrams are self-contained and render
identically in the draw.io desktop/web editor and in the exporter.

```powershell
# 1. (Re)generate the .drawio files from the Python definition
python build-diagrams.py

# 2. Render to PNG with the draw.io export Docker image (requires Docker Desktop running)
$dir = (Resolve-Path .).Path
docker run --rm -v "${dir}:/data" rlespinasse/drawio-export -f png --scale 2 --border 12

# 3. Copy the rendered PNGs into docs/images for embedding
Copy-Item .\export\architecture-Per-student-architecture.png ..\images\architecture.png -Force
Copy-Item .\export\access-flow-Access-flow.png ..\images\access-flow.png -Force
```

You can also open the `.drawio` files directly in <https://app.diagrams.net/> to edit them by
hand; the embedded icons travel with the file.

## Editing the layout

Prefer editing `build-diagrams.py` (it keeps coordinates, colors, and edge anchors in one
place) and regenerating, rather than hand-editing the `.drawio` XML. The `Diagram` helper class
exposes `container()`, `icon()`, `label()`, and `edge()` with explicit exit/entry anchors to
keep connectors tidy.
